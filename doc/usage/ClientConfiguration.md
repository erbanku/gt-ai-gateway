# 客户端接入配置指南

## 为什么需要客户端配置向导？ (背景介绍)

市面上有众多优秀的 AI 应用与编程助手，但它们接入第三方大模型网关的方式却千差万别。有些需要在界面的深层菜单中寻找 Base URL 和 API Key 的输入框，有些需要通过命令行环境变量进行覆盖，还有些会强制拼接默认的 API 路径从而导致连接失败。

为了解决这种繁杂的适配问题，避免用户到处翻找教程，GT AI Gateway 引入了**“客户端配置” (Client Configurations)** 功能模块。

该功能为您提供了一套智能向导，您只需选定目标客户端和网关鉴权令牌 (Token)，系统就会为您动态生成专属的接入说明、准确无误的 Base URL 和对应的 API Key。您不再需要去翻阅不同软件杂乱的设置项，只需跟着生成的步骤无脑复制粘贴，即可轻松将网关与您的开发工具无缝绑定。

## 使用步骤

### 1. 前置准备：创建供应商与用户
在进行客户端配置之前，您必须确保网关内已经添加了至少一个大模型供应商（Vendor）并创建了用来调用的用户（User/Token）。具体操作请参考前面的 [系统配置与使用指南](ConfigurationGuide.md)。

### 2. 进入客户端配置页面
在网关管理后台，点击左侧菜单栏的 **“客户端配置” (Clients)** 进入接入向导页面。

### 3. 选择您的客户端
在页面中您会看到多个以客户端名称命名的卡片。找到您正在使用或准备接入的客户端，点击该卡片上的 **“接入指南”** 按钮。

### 4. 选择使用的用户身份 (Token)
在弹出的配置对话框顶部，您需要选择一个用来鉴权的用户令牌（Tenant/Token）。
系统会基于您选择的用户 Token，自动生成专属的接口地址（Base URL）和 API Key。

### 5. 根据指南完成配置
仔细阅读对话框中生成的向导步骤。向导通常会告诉您：
1. 在客户端的哪个菜单里找到设置项。
2. 如何关闭默认请求。
3. 需要填入的 **Base URL** 是什么（可以直接一键复制）。
4. 需要填入的 **API Key** 是什么（可以直接一键复制）。
5. 如何覆盖或强制输入自定义模型名称。

### 6. 测试并重启
- 配置完成后，请在客户端发起一次简单的对话以测试连通性。
- 如果请求遇到问题，向导末尾通常会提供针对该客户端的故障排除建议（例如部分客户端需要彻底重启应用才能使环境变量或配置生效）。

## 协议与接口说明

网关支持标准的模型协议接口格式，其核心调用方式如下，如果您使用的客户端需要手动配置 URL，请参考：

- **OpenAI 协议接口**
  - **Base URL**: `http(s)://<您的网关域名>/v1`
  - **API Key**: 填入网关分配给您的用户令牌 (Token)

- **Anthropic 协议接口**
  - **Base URL**: `http(s)://<您的网关域名>`
  - **API Key**: 填入网关分配给您的用户令牌 (Token)

> 注意：网关具备 **自动协议转换** 功能，这意味着您完全可以在只支持 OpenAI 协议的客户端里输入 Anthropic 的模型名称（如 `claude-3-5-sonnet-latest`），网关会自动将其转换为目标模型格式。详见：[自动协议转换说明](ProtocolConversion.md)
